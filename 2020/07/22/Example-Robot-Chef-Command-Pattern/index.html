範例：機器人廚師（命令模式）

2020-07-22
Example

Pattern: 命令模式

Class Diagram: 機器人廚師

情境：這是一間牛排館，會根據客戶的訂單出菜

隨著生日蒸蒸日上，
老闆已經沒辦法同時處理客人點餐與煎牛排了（這是兩種職責）。

決定引進我們販賣的機器人廚師，
讓他能專心在記下客人的需求。

需求一：老闆決定到外場服務，透過指令指揮內場的機器人廚師煎牛排

  • 首先是機器人廚師的程式碼
  • 再定義出命令介面
  • 需要的指令：煎菲力牛排、煎沙朗牛排
  • 最後修改原本牛排館的程式

[角色對應關係]
調用者 (Invoker) ：老闆
接收者 (Receiver) ：機器人廚師
命令 (Command) ：煎牛排指令

[單一職責原則]
我們將點餐與煎牛排視作兩種不同的職責。
透過命令來聯繫兩者。

[開放封閉原則]
當新增/修改需求時，不會動到所有程式碼。
（比如：外場老闆不會知道烹調方式的改變、機器人不會知道當前餐廳的優惠）

[依賴反轉原則]
調用者依賴抽象的命令介面。
命令實作抽象的命令介面。

此外命令模式還可以留下點餐紀錄，便於將來結帳、重做餐點呢。

ʕ •ᴥ•ʔ：使用命令模式的老闆，也隱含「人力」資源管理的味道。
（指定誰做什麼餐點）
